# gateway_core.domain

## 1. 目录职责

- 领域无关小算法与学校业务通用解析能力。

## 2. 输入

- 用户问题、时间文本、业务条件。

## 3. 输出

- 规范化时间窗、过滤条件和领域辅助判断。

## 4. 核心文件

- `time_filter.py`：时间范围和学期/日期过滤解析。

## 5. 数据流

- Agent/SQL 工具调用领域解析函数，生成查询过滤条件或提示。

## 6. 不负责什么（Boundary）

- 不执行 SQL。
- 不调用模型。

## 7. 修改这里时的注意事项

- 保持纯函数优先；新增领域规则前先查 prompts/domain 配置是否已有。
