# gateway_core.logs

## 1. 目录职责

- 预留日志包/日志目录说明，不承载核心业务代码。

## 2. 输入

- 运行时日志、监控输出或 trace 文件引用。

## 3. 输出

- 供开发/运维查看的日志文件或说明。

## 4. 核心文件

- `README.md`：日志目录说明。

## 5. 数据流

- 运行时模块写入根目录 `logs/` 或配置的日志路径，本包只保留说明。

## 6. 不负责什么（Boundary）

- 不提交真实生产日志。
- 不放密钥、trace 原文大文件或数据库 dump。

## 7. 修改这里时的注意事项

- 新增日志类型时优先写到根 `logs/` 并加入 `.gitignore`。
