# gateway_core.agents.contracts

## 1. 目录职责

- 定义工具契约、规划结果和 Agent 调用约束。

## 2. 输入

- 工具描述、用户任务、规划请求。

## 3. 输出

- 结构化工具契约、Planner 输出和工具调用约束。

## 4. 核心文件

- `models.py`：契约数据模型。
- `planner.py`：契约规划。
- `tool_contract.py`：工具契约描述。

## 5. 数据流

- 上层 Agent 根据契约选择工具或构造受控调用。

## 6. 不负责什么（Boundary）

- 不执行工具。
- 不持有数据库连接或模型客户端。

## 7. 修改这里时的注意事项

- 契约模型变化会影响 Agent/tool 双方，必须同步测试。
